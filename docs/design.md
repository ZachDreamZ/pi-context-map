# pi-context-map Report — Design Specification

## 1. Purpose

The `pi-context-map` HTML report is generated by the `ReportGenerator` class and written to `~/.pi/context-map/report.html`. It visualizes:

- **Total token usage** and percentage of the 128k context window.
- **Context composition** (System / Tools / History / Files / Summaries) as a stacked bar.
- **Actionable insights** generated by the `InsightEngine` (tool bloat, stale files, high usage, etc.).
- **File-level detail** with status (Active / Stale / Legacy) and operation type (Read / Write / Edit).

The report is a **single static HTML file** with no external dependencies (no CDN, no fonts, no images). All styles and scripts are inlined.

## 2. Design Language

We adopt the **Linear design system** (referenced from `awesome-design-md/design-md/linear.app/`) for color, typography, and spacing. Card patterns follow **shadcn/ui** conventions (hairline borders, subtle elevation, neutral surfaces).

## 4. Interactivity (Vanilla JS)

The report includes a `<script>` block at the bottom with the following behaviors:

### 4.1 Collapsible Insights
Each insight card has a chevron toggle. Clicking collapses/expands the body. Default: all critical/warning expanded, info collapsed.

### 4.2 File Search
A search input above the file grid filters cards by path (case-insensitive substring match) in real-time.

### 4.3 Status Filter
A dropdown to filter files by status: All / Active / Stale / Legacy.

### 4.4 Composition Bar Tooltips
The stacked bar segments have `title` attributes for native tooltips (already present in v0.3.0).

## 5. Accessibility

- All interactive elements are keyboard-navigable (`<button>` elements).
- Color is not the only indicator: severity uses both color AND text labels (CRITICAL / WARNING / INFO).
- Sufficient contrast ratios maintained (WCAG AA: 4.5:1 for text).
- Semantic HTML: `<header>`, `<section>`, `<article>`, `<nav>`.

## 6. Non-Goals

- No external dependencies (no CDN, no Google Fonts).
- No build step (vanilla CSS/JS only).
- No React or framework — the report is a static document.
- No animations beyond simple CSS transitions (hover, expand/collapse).
